Compatible GPSses

gpsd should work with any GPS using an RS232C or USB interface that advertises NMEA-0183 compliance. Here are some notes on hardware we have tested (all under Linux 2.6.8). Hyperlinks lead to technical information.

Name Chipset Interface Sold as gpsd type Works with Sentences Notes
BU-303 SiRF-II USB NaviLock, Hyperdata, Globalsat, TripNav n 1.97 GGA, GSA, GSV, RMC, GLL, VTG Very generic SiRF II device. Requires pl2303 support which is broken in late 2.4 Linux kernels but good in 2.6.8. Here's a technical manual from Globalsat's U.S. reseller.
EarthMate Zodiac RS-232 DeLorme e 1.10 Unknown The old Zodiac version spoke Rockwell binary protocol. These models have been discontinued but are still common.
EarthMate SiRF-II USB DeLorme - None Unknown In 2003 the EarthMate switched to a SiRF-II chipset. The USB version doesn't work with gpsd under Linux, because support for the Cypress M8 serial-to-USB chipset is lacking. This is being worked on.
FV18 Unknown RS232 FV18 f 1.96 GGA, GSV, RMC, GLL, VTG (optionally GSA) Uses 8N2. Information on this unit is scant.
RoyalTek Sapphire USB SiRF-II USB   n 1.97 GGA, GSV, RMC, GLL, VTG, GSA There's an RS232 variant as well, not yet tested.
TripMate Zodiac RS-232 Delorme t 1.97 GPRMC, GPGSA, GPGGA, GPGSV, PRWIRID, PRWIZCH Discontinued sometime before November 1998. Takes optional latitude/longitude initialization.
